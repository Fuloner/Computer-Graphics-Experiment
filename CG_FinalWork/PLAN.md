# PLAN.md —— 基于PBF的3D流体模拟开发流程

## 第1步：基础框架搭建（已完成）
**目标：** 搭建OpenGL渲染环境，实现自由摄像机控制。

- 创建 `MyInit.h`：封装GLFW窗口初始化、OpenGL上下文创建、回调函数设置
- 创建 `MyCamera.h`：实现第一人称自由摄像机（WASD移动 + 鼠标旋转 + 滚轮缩放），支持俯仰/偏航角限制，使用glm::lookAt计算观察矩阵
- 创建 `MyShader.h`：封装着色器的编译、链接、激活、uniform设置等功能
- 编写一个简单的测试着色器，在 `main.cpp` 中渲染一个基础的三角形或立方体，验证窗口、摄像机、着色器三个模块均能正常工作
- **你看到的效果：** 一个深色背景的窗口，可以用鼠标和键盘自由旋转/移动视角，窗口中渲染一个简单的3D物体（如线框立方体），按ESC退出

## 第2步：粒子系统基础（已完成）
**目标：** 定义流体粒子的数据结构，在3D空间中初始化一"块"粒子，并用点精灵（GL_POINTS）渲染它们。

- 定义 `Particle` 结构体：包含位置(glm::vec3)、速度(glm::vec3)、预测位置(glm::vec3)、lambda(标量，拉格朗日乘子)
- 定义 `FluidSystem` 类，管理所有粒子的数组
- 初始化粒子：在水箱上半部分以立方体晶格排列生成粒子（如10×10×10），给每个粒子设置相同的质量
- 在顶点着色器内部将点精灵大小与屏幕空间关联（gl_PointSize随距离缩小），片元着色器中绘制圆形点（用length(gl_PointCoord)剔除圆外像素），赋予浅蓝色
- 在当前帧中只渲染粒子位置（点精灵），不做任何物理计算
- **你看到的效果：** 3D空间中悬浮着一个由蓝色小圆点组成的方块形状，可以用摄像机环绕观察

## 第3步：邻居搜索（空间哈希）（已完成）
**目标：** 实现高效的空间哈希网格，让每个粒子能快速找到距离自己小于光滑核半径h的所有邻居粒子。

- 定义 `SpatialHashing` 类：
  - 将3D空间划分成边长为h的立方体网格单元
  - 提供 `hashPosition(glm::vec3)` 函数：根据位置坐标计算对应的哈希值（网格单元ID）
  - 提供 `build()` 函数：遍历所有粒子，将粒子索引存入对应哈希桶
  - 提供 `findNeighbors(glm::vec3 pos)` 函数：查询该位置周围3×3×3=27个相邻网格单元中的所有粒子，筛出距离 < h的粒子作为邻居
- 在 `FluidSystem` 中集成空间哈希，编写测试：对每个粒子，打印邻居数量（或直接在渲染中，把有邻居的粒子显示为白色、无邻居的显示为蓝色）
- **你看到的效果：** 仍是一方块蓝色粒子，现在每个粒子找到了邻近的粒子（内部的粒子会有非零邻居数），可以从控制台输出看到邻居统计

## 第4步：PBF核心算法 —— 密度约束求解
**目标：** 实现PBF论文中最核心的"不可压缩性"约束 —— 通过迭代修正粒子位置，迫使每个粒子处的密度等于静止密度ρ₀。

这一步的工作分为4个子部分，都在 `FluidSystem::solveDensityConstraint()` 中完成：

### 4a：密度估算
对每个粒子i，用Poly6核函数计算其密度：
```
ρ_i = Σ m_j * W_poly6(p_i - p_j, h)
```
其中W_poly6是光滑核函数，h是光滑核半径，求和遍历所有邻居j（包括自身）。

### 4b：约束函数与拉格朗日乘子
约束函数定义为：`C_i(p) = ρ_i / ρ_0 - 1`（密度等于静止密度时约束为0）。

对每个粒子i，计算其lambda值：
```
λ_i = -C_i / ( Σ_k |∇_p_k C_i|² + ε )
```
其中梯度∇_p_k C_i需要分k=i和k=j（邻居）两种情况推导，ε用于避免除零。

### 4c：位置修正
对每个粒子i，遍历邻居j（包括自身），计算总的位置修正量：
```
Δp_i = (1/ρ_0) * Σ_j (λ_i + λ_j + s_corr) * ∇W_spiky(p_i - p_j, h)
```
其中s_corr是人工压力项（用于避免粒子聚集），用Spiky核函数的梯度。

### 4d：迭代求解
将步骤4a-4c重复执行若干次（如3-5次），每次都将粒子预测位置p_i*加上修正量Δp_i，使密度约束逐渐被满足。

- 先在GPU之外、CPU端（FluidSystem中）实现上述所有计算逻辑
- **你看到的效果：** 粒子方块开始坍塌并扩散 —— 原本规则的晶格排列在重力（见下一步） + 密度约束作用下变得不规则，粒子向外流动，形成类似液滴摊开的形状

## 第5步：外力、碰撞与速度更新
**目标：** 加入重力、水箱边界碰撞、速度更新和粘性，让模拟成为完整的"流体动画"。

- **外力（重力）：** 每一帧开始时，为每个粒子的速度加上 `g * Δt`（g取(0, -9.8, 0)），然后预测位置 `p* = p + v * Δt`
- **边界碰撞：** 在水箱六个面上做碰撞检测与响应：
  - 如果粒子预测位置超出水箱边界，将位置修正回边界上（可加一点反弹系数），并将速度在该方向的分量置零或反向衰减
  - 水箱尺寸建议：宽×高×深 = 1.0×2.0×1.0（自然单位）
- **速度更新：** 约束求解后，用最终位置反推速度：`v_i = (p_i_final - p_i_original) / Δt`
- **XSPH粘性：** 对速度做一次额外的平滑处理，用Poly6核对邻居速度加权平均，使流体流动更自然、不出现剧烈抖动
- **集成到FluidSystem：** 将这些步骤组合成每帧的模拟循环：外力 → 预测位置 → 邻居搜索 → 密度约束迭代 → 速度更新 → XSPH粘性
- **你看到的效果：** 粒子方块受重力下落，撞击水箱底部后溅开，最终在水箱底部形成一层相对平静的水面 —— 真正的流体动画！

## 第6步：流体渲染（屏幕空间 + 光照）
**目标：** 将粒子渲染成"像水一样"的半透明球体，而不是简陋的圆点。

- **方案选择：** 使用实例化渲染（glDrawArraysInstanced），每个粒子渲染一个小球体（低多边形icosphere或使用模型矩阵放缩+点精灵增强），配合一个流体专用的着色器
- **着色器设计：**
  - 顶点着色器：用粒子位置构建模型矩阵，缩放为粒子半径（约h/2），传递法线
  - 片元着色器：实现简化的Blinn-Phong光照（漫反射 + 镜面高光 + 环境光），水面颜色为半透明浅蓝，alpha值根据粒子密度调整
- **混合设置：** 开启OpenGL混合（glEnable(GL_BLEND)），使用 `GL_SRC_ALPHA, GL_ONE_MINUS_SRC_ALPHA`，渲染顺序上先渲染不透明物体（水箱），再渲染半透明粒子
- **屏幕空间厚度（可选增强）：** 如果需要更好的流体视觉，可渲到FBO中积累粒子"厚度"，再与背景混合
- **你看到的效果：** 不再是离散的圆点，而是一团带有光泽、半透明的蓝色流体，可以看到"水面"的高光和下层物体的颜色穿透

## 第7步：水箱渲染与交互完善
**目标：** 渲染透明的3D包围盒（水箱），完善用户交互。

- **水箱渲染：** 用线条（GL_LINES）渲染水箱的12条棱，颜色为白色半透明。或用半透明面绘制6个面（开启面剔除确保从外部也能看到内部粒子）
- **默认视角：** 程序启动时，摄像机置于水箱前方适当距离，可以在代码中设置一个合适的水箱世界空间位置（如圆心在原点上方）
- **可调参数面板（简单）：** 在终端中输出按键提示，支持按键切换：
  - R键：重置模拟
  - 空格键：暂停/继续模拟
  - P键：在当前鼠标方向释放一"团"新粒子（模拟注水效果）
  - 数字键1-3：切换不同颜色的流体
- **帧率显示：** 在终端或窗口标题中输出当前的FPS和粒子数量
- **你看到的效果：** 完整的演示程序 —— 透明水箱、半透明蓝色流体动画、可通过按键交互，FPS实时显示在标题栏
